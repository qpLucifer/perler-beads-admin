# 拼豆 DIY 管理后台（React + Ant Design）

本项目是拼豆 DIY 的运营与管理端，负责用户、商品、订单、模板、作品、评价等后台管理功能。系统与后端共用业务数据，并与小程序端保持功能与字段一致性。

## 1. 项目定位

- 面向管理员与运营角色。
- 提供可视化管理能力（列表筛选、详情查看、状态更新、批量操作）。
- 提供数据与调试增强能力（筛选修正统计、事件跳转排查链路）。

## 2. 技术栈

- React + Vite
- Ant Design
- React Router
- Axios
- Day.js

## 3. 主要功能模块

- 仪表盘：核心统计、最近订单、筛选修正统计面板。
- 用户管理：分页、搜索、状态筛选、编辑、重置密码。
- 商品管理：分类筛选、排序、增删改。
- 订单管理：状态筛选、金额排序、详情查看、状态流转。
- 作品管理：搜索、排序、预览、删除。
- 模板管理：高级筛选、排序、预览、批量删除、批量官方/用户切换。
- 评价管理：评价列表与删除能力。

## 4. 近期增强（已内置）

- 统一列表状态管理（URL + localStorage 双持久化）。
- 统一“重置筛选”能力。
- 支持服务端 `applied_filters` 对齐：
  - 发现修正时提示
  - 自动同步前端筛选状态
- Dashboard 新增筛选修正统计：
  - 全部 / 近 7 天视图
  - 按页面/按字段聚合
  - 最近修正事件（最新 10 条）
  - 导出 JSON、清空统计
  - 一键跳转到对应列表页并聚焦字段

## 5. 快速开始

### 5.1 安装依赖

```bash
npm install
```

### 5.2 本地开发

```bash
npm run dev
```

默认访问：`http://localhost:5173`

### 5.3 构建与预览

```bash
npm run build
npm run preview
```

## 6. 运行依赖

- 需要 `perler-beads-backend` 先启动并可访问。
- 默认 API 基地址配置在 `src/api/index.js`：
  - `http://localhost:3000/api`

如后端地址变化，请同步修改该配置。

## 7. 目录结构（核心）

```text
perler-beads-admin/
├─ src/
│  ├─ api/                   # 接口封装与拦截器
│  ├─ layouts/               # 主布局
│  ├─ pages/                 # 业务页面
│  ├─ hooks/                 # 通用 hooks（如 useListState）
│  ├─ utils/                 # 工具函数（鉴权、筛选统计、聚焦定位等）
│  ├─ App.jsx
│  └─ main.jsx
├─ package.json
└─ README.md
```

## 8. 认证与权限

- 登录成功后保存 `admin_token`。
- 请求拦截器自动注入 `Authorization`。
- 遇到 `401` 自动清 token 并跳转登录。
- 管理权限在后端校验，前端主要做路由保护和交互提示。

## 9. 常见问题排查

- 无法登录：检查后端是否启动、管理员账号是否存在。
- 列表无数据：先看网络请求，再看后端 `applied_filters` 是否修正了参数。
- 筛选值异常：Dashboard 的筛选修正统计可快速定位问题字段。
- 页面跳转但未高亮：检查 URL 是否包含 `focus_field`。
